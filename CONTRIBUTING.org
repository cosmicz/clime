#+title: Contributing

* Getting Started

1. Fork and clone the repository
2. Run ~make test~ to ensure everything passes
3. Create a feature branch (~feature/your-feature~ or ~fix/your-fix~)

* Development

** Running Tests

#+begin_src sh
make test                                  # Run all tests
make test SELECT="^clime-test-core"     # Run specific tests
make compile                               # Byte-compile
make clean                                 # Clean artifacts
#+end_src

** Code Style

- Use ~lexical-binding: t~ in all files
- Prefix all symbols with ~clime-~ (public) or ~clime--~ (internal)
- Write docstrings for all public functions
- Follow standard Emacs Lisp conventions

** Commit Messages

- Use present tense ("Add feature" not "Added feature")
- Keep the first line under 72 characters
- Reference issues when applicable

* Submitting Changes

1. Ensure all tests pass (~make test~)
2. Byte-compile cleanly (~make compile~)
3. Open a pull request with a clear description
