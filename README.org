#+title: clime
#+options: toc:nil num:nil

Declarative CLI framework for Emacs Lisp.  Build command-line tools with
subcommands, options, flags, and help generation --- all from a single
~clime-app~ declaration.

* Features

- Declarative DSL: define commands, options, and args in one form
- Subcommands and nested groups (e.g. ~myapp repo add~)
- Auto-generated ~--help~ and ~--version~
- Boolean flags, count flags (~-vvv~), multiple values (~--tag a --tag b~)
- Environment variable support with auto-derived names
- Stdin reading via ~-~ sentinel
- JSON output mode
- Type coercion (string, integer, number)
- Single-file vendorable distribution via ~bundle~ command

* Installation

** Git clone / submodule

#+begin_src bash
# Clone directly
git clone https://github.com/cosmicz/clime vendor/clime

# Or as a submodule
git submodule add https://github.com/cosmicz/clime vendor/clime
#+end_src

Then initialize your app file:

#+begin_src bash
vendor/clime/clime-app.el init myapp.el
#+end_src

This adds a polyglot shebang header and makes the file executable.

** package-vc (Emacs 29+)

#+begin_src emacs-lisp
(package-vc-install "https://github.com/cosmicz/clime")
#+end_src

** Vendoring (single file)

Build a single-file bundle and copy it into your project:

#+begin_src bash
cd clime/
make dist
cp dist/clime.el /path/to/myapp/
#+end_src

Your app just needs ~(require 'clime)~ and ~-L~ pointing to the
directory containing the bundled file.

* Getting Started

** Create your app file

Write a standard Emacs Lisp file with the clime DSL:

#+begin_src emacs-lisp
;;; greet.el --- A greeting CLI  -*- lexical-binding: t; -*-

;;; Code:

(require 'clime)

(clime-app greet
  :version "1.0.0"
  :help "A friendly greeter."

  (clime-option shout ("--shout" "-s") :flag t
    :help "SHOUT THE GREETING")

  (clime-command hello
    :help "Say hello to someone"
    (clime-arg name :help "Person to greet")
    (clime-handler (ctx)
      (clime-let ctx (name shout)
        (let ((msg (format "Hello, %s!" name)))
          (if shout (upcase msg) msg)))))

  (clime-command goodbye
    :help "Say goodbye"
    (clime-arg name :help "Person to bid farewell")
    (clime-handler (ctx)
      (clime-let ctx (name)
        (format "Goodbye, %s!" name)))))

(clime-run-batch greet)

(provide 'greet)
;;; greet.el ends here
#+end_src

Then initialize it with a shebang:

#+begin_src bash
path/to/clime-app.el init greet.el
#+end_src

** Run it

#+begin_src bash
./greet.el hello world        # => Hello, world!
./greet.el --shout hello world # => HELLO, WORLD!
./greet.el --help
./greet.el hello --help
#+end_src

** Add more features

*Environment variables* --- set ~:env-prefix~ on the app to auto-derive
env var names from options:

#+begin_src emacs-lisp
(clime-app greet
  :version "1.0.0"
  :help "A friendly greeter."
  :env-prefix "GREET"
  ...)
#+end_src

Now ~--shout~ can also be set via ~GREET_SHOUT=1~.

*Nested groups* --- organize related commands:

#+begin_src emacs-lisp
(clime-group templates
  :help "Manage greeting templates"

  (clime-command list
    :help "List available templates"
    (clime-handler (_ctx) "default, formal, casual"))

  (clime-command add
    :help "Add a template"
    (clime-arg name :help "Template name")
    (clime-arg body :help "Template body")
    (clime-handler (ctx)
      (clime-let ctx (name body)
        (format "Added template %s: %s" name body)))))
#+end_src

*Stdin* --- use ~-~ as an argument value to read from stdin:

#+begin_src bash
echo "World" | ./greet.el hello -
#+end_src

* Bundling & Distribution

The ~clime-app.el~ tool provides two commands for distribution:

*~bundle~* --- concatenate multiple source files into a single =.el=:

#+begin_src bash
./clime-app.el bundle -o dist/myapp.el \
    --provide myapp --main myapp \
    src/myapp-core.el src/myapp-commands.el src/myapp.el
#+end_src

The ~--main APP~ flag adds a guarded entry point so the bundle works
as both a library (~require 'myapp~) and a CLI.

*~init~* --- add a shebang to make any =.el= file executable:

#+begin_src bash
./clime-app.el init --env CLIME_MAIN_APP=myapp dist/myapp.el
#+end_src

For dual-use bundles, pass ~--env CLIME_MAIN_APP=<app>~ so the
guarded entry point activates only when executed directly.

* Project Structure

A recommended layout for clime-based apps:

#+begin_example
myapp/
  myapp.el              # main app with shebang
  myapp-commands.el     # optional: split commands into modules
  Makefile              # optional: bundle + init targets
  tests/
    myapp-tests.el
#+end_example

* DSL Reference

| Macro            | Purpose                            | Key arguments                                          |
|------------------+------------------------------------+--------------------------------------------------------|
| ~clime-app~      | Define a CLI application           | ~:version~, ~:help~, ~:env-prefix~                     |
| ~clime-command~  | Define a subcommand                | ~:help~, ~:aliases~, ~:hidden~                         |
| ~clime-group~    | Nest related commands              | ~:help~, ~:aliases~, ~:hidden~                         |
| ~clime-option~   | Declare a named option             | ~:flag~, ~:count~, ~:multiple~, ~:required~, ~:type~   |
| ~clime-arg~      | Declare a positional argument      | ~:nargs~, ~:type~, ~:default~, ~:help~                 |
| ~clime-handler~  | Attach a handler to a command      | ~(ctx)~ — receives the parse context                   |
| ~clime-let~      | Destructure context in a handler   | ~ctx (var1 var2 ...)~ — binds option/arg values        |

See =examples/pkm.el= for a comprehensive demo covering all major features.
